## Custom Project

### Dataset

The dataset file used was `sales_Asher.csv`.

The dataset contains sales transaction records with order, product, customer, and payment information.

I modified the original sales dataset by adding additional records.

### Kafka Messages

The producer sends one sales record at a time to Kafka.

The message key used was `region_id` and the fields remained unchanged.

### Consumer Processing

The consumer receives sales records from Kafka and processed 36 messages.

It logs sales statistics, calculates totals, and writes records to `consumed_sales_Asher2.csv`.

### Experiments

For Phase 4, I changed the chart title and axis labels to make the visualization more descriptive.

For Phase 5, I changed the chart from a line graph to a bar graph using `axis.bar()`.

### Results

The producer successfully streamed records and the consumer processed all messages.

The consumer saved results to a CSV file, DuckDB database, and live chart.

### Interpretation

I learned how messages move through Kafka from the producer to the consumer.

The stream provided useful business intelligence such as total sales, average sales, and sales trends.
